# 联网搜索功能接入指南

> 让 Karvis 能搜索互联网，回答实时问题（天气、新闻、知识百科、技术文档等）。
>
> 基于腾讯云 lkeap 原生联网搜索能力，**不需要额外 API Key，不需要新依赖**，改 4 个文件就能用。

---

## 原理

腾讯云知识引擎 lkeap 的 DeepSeek API 支持一个 `enable_search` 参数。开启后，lkeap 会在调用大模型之前先走搜狗搜索引擎获取实时信息，然后让模型基于搜索结果生成回答。

对 Karvis 来说，就是新增一个 `web.search` Skill：

```
用户: "最近有什么好看的电影？"
  ↓
brain.py → LLM 决策: skill="web.search", params={query: "2026年热门电影推荐"}
  ↓
web_search.py → 调 lkeap API（enable_search=True）→ 搜索+回答一步完成
  ↓
回复: "最近口碑比较好的有..."
```

整个过程只有一次 API 调用，和普通对话差不多快。

---

## 前置条件

在腾讯云 lkeap 控制台开启联网搜索的计费开关：

👉 https://console.cloud.tencent.com/lkeap/settings

找到「联网搜索」，开启后付费开关即可。按次计费，很便宜。

> 如果不开这个开关，`enable_search` 参数会被忽略，不会报错但也不会搜索。

## 第 1 步：新建 `src/skills/web_search.py`

在 `src/skills/` 目录下创建 `web_search.py`，完整内容如下：

```python
# -*- coding: utf-8 -*-
"""
Skill: web.search
联网搜索 — 让 Karvis 能搜索互联网获取实时信息。

实现方式：
  腾讯云知识引擎 lkeap 原生联网搜索（enable_search=True）
  - 无需额外 API Key，复用已有的 DEEPSEEK_API_KEY
  - 无需额外依赖，复用已有的 requests
  - 搜索 + 回答一步完成，lkeap 内部调搜狗搜索引擎
"""
import sys
import requests
from datetime import datetime, timezone, timedelta

BEIJING_TZ = timezone(timedelta(hours=8))


def _log(msg):
    ts = datetime.now(BEIJING_TZ).strftime("%H:%M:%S")
    print(f"{ts} {msg}", file=sys.stderr, flush=True)


def execute(params, state, ctx):
    """
    web.search — 联网搜索（腾讯云 lkeap 原生能力）

    params:
        query: str — 搜索关键词/问题
        detailed: bool — 是否需要详细结果（默认 false）

    returns:
        {"success": bool, "reply": str}
    """
    from config import DEEPSEEK_API_KEY, DEEPSEEK_BASE_URL, DEEPSEEK_MODEL

    query = (params.get("query") or "").strip()
    detailed = params.get("detailed", False)

    if not query:
        return {"success": False, "reply": "想搜什么呢？告诉我你想查的内容~"}

    _log(f"[web.search] 开始联网搜索: query={query}")

    # ── 调用 lkeap DeepSeek API，开启 enable_search ──
    url = f"{DEEPSEEK_BASE_URL}/chat/completions"
    headers = {
        "Authorization": f"Bearer {DEEPSEEK_API_KEY}",
        "Content-Type": "application/json",
    }

    system_msg = (
        "你是用户的 AI 助手，用户需要你联网查找信息。"
        "请根据搜索到的最新信息，用自然简洁的语言回答。"
        "回答要求：\n"
        "1. 直接回答，不要说"根据搜索结果"之类的话\n"
        "2. 信息准确，必要时标注来源\n"
        "3. 控制在 300 字以内，除非用户要求详细\n"
        "4. 口语化，像朋友聊天"
    )
    if detailed:
        system_msg += "\n5. 用户希望了解详细信息，可以多写一些，分点说明。"

    data = {
        "model": DEEPSEEK_MODEL,
        "messages": [
            {"role": "system", "content": system_msg},
            {"role": "user", "content": query},
        ],
        "enable_search": True,       # 🔑 关键：腾讯云 lkeap 联网搜索开关
        "max_tokens": 800 if detailed else 500,
        "temperature": 0.3,
    }

    try:
        t0 = _now_ts()
        resp = requests.post(url, headers=headers, json=data, timeout=30)
        elapsed = _now_ts() - t0

        if resp.status_code == 200:
            result = resp.json()
            answer = result["choices"][0]["message"]["content"].strip()

            # 提取搜索引用信息（如果有）
            search_results = result["choices"][0]["message"].get("search_results")
            source_hint = ""
            if search_results:
                sources = []
                for sr in search_results[:3]:
                    name = sr.get("name") or sr.get("title", "")
                    if name:
                        sources.append(name)
                if sources:
                    source_hint = "\n\n📎 参考：" + "、".join(sources)

            _log(f"[web.search] 联网回答完成: {elapsed:.1f}s, {len(answer)} chars"
                 f", 引用{len(search_results or [])}条")

            return {"success": True, "reply": answer + source_hint}

        _log(f"[web.search] API 错误: {resp.status_code} - {resp.text[:200]}")
        return {"success": False, "reply": "搜索出了点问题，稍后再试试~"}

    except requests.exceptions.Timeout:
        _log(f"[web.search] 请求超时")
        return {"success": False, "reply": "搜索超时了，网络可能不太好，再试一次？"}
    except Exception as e:
        _log(f"[web.search] 异常: {e}")
        return {"success": False, "reply": "搜索遇到问题了，稍后再试~"}


def _now_ts():
    import time
    return time.time()


# ============ Skill 热加载注册表 ============
SKILL_REGISTRY = {
    "web.search": execute,
}
```

> **为什么放进去就能用？** `skill_loader.py` 会自动扫描 `src/skills/` 目录下所有 `.py` 文件，读取其中的 `SKILL_REGISTRY` 字典并注册。不需要手动 import。

## 第 2 步：修改 `src/prompts.py`（2 处）

### 2.1 在 `SKILL_PROMPT_LINES` 字典中注册 Skill 描述

找到 `SKILL_PROMPT_LINES` 字典，在 `"ignore"` 那行**前面**加一行：

```python
    # ↓↓↓ 新增这一行 ↓↓↓
    "web.search": '**web.search** `{query, detailed?}` — 联网搜索：查询实时信息、新闻、知识百科、技术文档等。query=搜索关键词（请优化为适合搜索引擎的关键词）；detailed=true 时返回更多结果。当用户问你不确定的事实、需要实时数据（天气/新闻/股价/赛事）、或明确要求"帮我查一下"时使用。',
    "ignore": '**ignore** `{reason?}` — 不处理',
```

**这一步的作用**：让 LLM 在决策时知道有 `web.search` 这个 Skill 可以选。

### 2.2 在 `RULES_CORE` 中添加触发规则

找到 `RULES_CORE` 字符串里的 `## 闲聊与日常互动` 这一行，在它**前面**加一段：

```
## 联网搜索
- 用户问实时信息（天气/新闻/股价/赛事/热搜/最近的XX） → `web.search`
- 用户问你不确定的事实、人物、事件 → `web.search`（别瞎编，搜一下）
- 用户明确说"帮我查一下"、"搜一下"、"上网看看" → `web.search`
- 用户问技术问题/API用法/最新版本特性 → `web.search`
- query 参数请优化为适合搜索引擎的关键词，比如用户说"最近有什么好电影"→ query="2026年3月热门电影推荐"
- **不要搜的情况**：用户个人数据（笔记/待办/记忆） → 用 internal.search / deep.dive；纯闲聊/情感倾诉 → ignore 正常回复
```

**这一步的作用**：告诉 LLM 什么时候该选 `web.search`、什么时候不该选。

## 工作原理（给想深入理解的人）

### Skill 体系回顾

```
用户消息 → brain.process()
  → 构建 system prompt（包含 SKILL_PROMPT_LINES + RULES）
  → 调 LLM → 返回 JSON 决策: {skill: "web.search", params: {query: "..."}}
  → skill_loader 查找 handler → 执行 web_search.execute()
  → 返回 {reply: "..."}
```

### 联网搜索的关键

核心就是 API 请求里多了一个字段：

```python
data = {
    "model": "deepseek-chat",
    "messages": [...],
    "enable_search": True,   # ← 就这一个参数
}
```

lkeap 收到后：
1. 先把用户问题扔给搜狗搜索，拿到网页结果
2. 把搜索结果注入到 prompt context 中
3. 让 DeepSeek 基于搜索结果生成回答
4. 返回结果中附带 `search_results` 字段（引用来源）

### 和 brain.py 里普通 LLM 调用的区别

| | brain.py 普通调用 | web_search.py 联网调用 |
|--|---|---|
| `enable_search` | 无此参数 | `True` |
| 知识范围 | 模型训练数据 | 训练数据 + 实时搜索 |
| 响应时间 | 2-5s | 3-8s（多了搜索耗时） |
| 费用 | 纯 LLM token | LLM token + 搜索费用 |

### Handler 签名

所有 Skill handler 统一签名：

```python
def execute(params: dict, state: dict, ctx: UserContext) -> dict:
    # params: LLM 决策时填的参数（如 {query: "...", detailed: true}）
    # state:  当前用户状态（短期记忆、打卡状态等）
    # ctx:    用户上下文（IO 接口、文件路径等）
    return {"success": True, "reply": "回复内容"}
```

## 常见问题

**Q: 搜索没触发，AI 还是在瞎编？**
- 检查 `prompts.py` 的 `SKILL_PROMPT_LINES` 里有没有加 `web.search` 那一行
- 检查 `RULES_CORE` 里有没有加「联网搜索」那段规则
- 重启服务让 Skill 重新加载

**Q: 报错 `enable_search` 无效？**
- 去 lkeap 控制台检查联网搜索开关是否开启：https://console.cloud.tencent.com/lkeap/settings

**Q: 可以给所有对话都开联网吗？**
- 技术上可以（在 `brain.py` 的 `_call_deepseek` 里加 `enable_search=True`），但不建议——搜索有额外费用和延迟，大部分聊天不需要联网。通过 Skill 按需触发是更合理的方式。

**Q: 想换别的搜索引擎（比如 Tavily、Google）？**
- 改 `web_search.py` 里的搜索逻辑即可，handler 签名和返回格式不变，上层完全无感。
