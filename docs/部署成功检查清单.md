# 部署成功检查清单

> 部署完对照勾选，全部通过 = 可以正常使用。

---

## 基础服务

- [ ] **Karvis 服务正在运行**
  - 访问 `http://localhost:9000` 有响应（不是超时）
  - 日志中没有启动报错

- [ ] **公网可以访问**
  - 在浏览器中打开 `http://你的IP:9000/web/login`（云服务器）
  - 或打开 `https://xxx.trycloudflare.com/web/login`（cloudflared）
  - 能看到登录页面

---

## 企微连接

- [ ] **回调 URL 保存成功**
  - 企微后台 → 应用 → 接收消息 → URL 已保存，没有报错

- [ ] **Token 和 AESKey 一致**
  - `.env` 中的 `WEWORK_TOKEN` 和企微后台填的一样
  - `.env` 中的 `WEWORK_ENCODING_AES_KEY` 和企微后台填的一样

- [ ] **企业可信 IP 已配置**（云服务器必做，cloudflared 跳过）
  - 应用详情 → 企业可信 IP → 已填入服务器公网 IP

---

## 消息收发

- [ ] **发"你好"收到回复**
  - 在微信/企微中找到 Karvis，发一句"你好"
  - 收到欢迎消息或正常回复

- [ ] **日志中能看到消息记录**
  - 管理端 `/web/logs` 或命令行日志中搜索 `[handle_message]`
  - 能看到你发送的消息

---

## 管理端

- [ ] **管理端可以访问**
  - 浏览器打开 `http://你的IP:9000/web/logs`
  - 用 `ADMIN_TOKEN` 登录成功

---

## 定时任务（可选验证）

- [ ] **早报推送正常**
  - 等待次日早上 8 点左右，检查是否收到早报
  - 或手动触发：
    ```bash
    curl -X POST http://localhost:9000/system \
      -H "Content-Type: application/json" \
      -d '{"action":"morning_report"}'
    ```
