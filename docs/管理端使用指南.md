# 管理端使用指南

> Karvis 内置了一个 Web 管理端，可以查看日志、管理用户、监控 LLM 用量。

---

## 访问管理端

### 地址格式

```
http://你的IP:9000/web/logs
```

示例：
- 云服务器：`http://43.138.xx.xx:9000/web/logs`
- 本地部署：`http://127.0.0.1:9000/web/logs`
- cloudflared：`https://xxx-xxx.trycloudflare.com/web/logs`

### 登录

打开地址后会要求输入密码，填写你在 `.env` 中设置的 `ADMIN_TOKEN` 值。

> 忘了 ADMIN_TOKEN？打开 `src/.env` 文件查看。如果用 setup.sh 部署，密码是自动生成的随机字符串。

## 管理端功能

### 日志查看（/web/logs）

管理端主页，包含三个 Tab：

**日志 Tab**：
- 实时查看 Karvis 的运行日志
- 可以看到用户消息收发、AI 处理过程、错误信息

**统计监控 Tab**：
- LLM 调用次数和 token 用量
- 延迟瀑布图（可点击展开详情）
- Prompt token 分布直方图
- 成本预算进度

**错误聚合 Tab**：
- 自动提取 ERROR 和 Traceback
- 按错误类型去重计数
- 类似 Sentry 的 Top 20 错误展示

### 管理后台（/web/admin）

用户运营中心：
- 用户列表：查看所有注册用户
- 用户状态：活跃/不活跃/挂起
- LLM 用量：每用户的 token 消耗
- 操作：挂起/激活用户

### 用户 Web 端（/web/overview）

用户自己查看数据的页面，共 14 个页面：
- 概览、速记、待办、日记、笔记、情绪、记忆、习惯、决策、反思、设置等

用户通过发送"查看"或"web"给 Karvis，获取带 token 的链接访问。

## 日志排查

### 常用排查方法

**在管理端查看**：
打开 `/web/logs`，日志 Tab 中搜索关键词。

**在服务器命令行查看**：

```bash
# Docker 部署
docker logs -f karvis                    # 实时日志
docker logs karvis --tail 100            # 最近 100 行

# systemd 部署
journalctl -u karvis -f                  # 实时日志
journalctl -u karvis --since "1 hour ago"

# nohup 部署
tail -f karvis.log                       # 实时日志
```

### 常见日志关键词

| 搜索关键词 | 含义 |
|-----------|------|
| `[handle_message]` | 收到用户消息 |
| `[Brain]` | AI 正在处理 |
| `reply_text` | 发送回复 |
| `IP not in whitelist` | 可信 IP 没配 |
| `error` / `Error` | 各种错误 |
| `balance` / `余额` | API 余额问题 |

### 手动触发定时任务

```bash
# 触发晨报
curl -X POST http://localhost:9000/system \
  -H "Content-Type: application/json" \
  -d '{"action":"morning_report"}'

# 触发日报
curl -X POST http://localhost:9000/system \
  -H "Content-Type: application/json" \
  -d '{"action":"daily_report"}'

# 刷新缓存
curl -X POST http://localhost:9000/system \
  -H "Content-Type: application/json" \
  -d '{"action":"refresh_cache"}'
```
